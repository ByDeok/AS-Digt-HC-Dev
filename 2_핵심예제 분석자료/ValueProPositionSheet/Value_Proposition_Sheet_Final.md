### 2) JTBD 관점 Job/Goal 정리 (JTBD 선언)

(출처: `8_JTBD 계획서`, `9_JTBD 가상 결과`, `6_여정지도`, `7_AOS-DOS`)

- **Job 1 – 진료 전 데이터 정리·공유 Job**  
  - When: 정기 검진·외래 진료를 앞두고 집에서 측정한 혈압·혈당·운동 기록과 병원 결과지를 들고 진료실에 들어갈 때  
  - I want to: **여러 앱·기기 데이터를 1장 요약 리포트로 정리**해 의사에게 신뢰 있게 보여주고  
  - So I can: 5분 내에 상태·조치사항을 명확히 이해하고, 추가 검사·약 조정을 **한 번에 결정**하고 싶다.

- **Job 2 – 행동 변화 코칭 Job**  
  - When: 워치·혈압계에서 수치가 나쁘게 나오거나 복약·운동을 놓친 날  
  - I want to: 복잡한 차트 대신 **“오늘 해야 할 1~3개 행동”**만 조용히 제안받아서  
  - So I can: 불안·죄책감을 줄이고, 작은 행동으로 **실제 변화를 만드는 감각**을 갖고 싶다.

- **Job 3 – 가족·보호자 협업 Job**  
  - When: 부모의 건강 데이터를 대신 챙기고, 예약·문자·앱 알림을 매번 정리해야 할 때  
  - I want to: 각자 계정이더라도 **하나의 가족 보드**에서 약·일정·위험 알림을 함께 보고  
  - So I can: 돌봄이 특정 가족에게 몰리지 않고 **지속 가능한 역할 분담 구조**를 만들고 싶다.

- **Job 4 – 3분 온보딩/도입 장벽 제거 Job**  
  - When: 새로운 건강관리 앱이나 병원 연동 서비스를 시작하려고 할 때  
  - I want to: 가입·본인인증·기기연동·병원연동까지 **3분 안에 한 번에 끝내고**  
  - So I can: “나중에 하자”로 미루지 않고, **첫날 핵심 기능까지 바로 도달**하고 싶다.

### 4) 우리 솔루션의 핵심 제안 (Value Proposition)

(출처: `3_KSFs`, `5_TAM-SAM-SOM`, `6_페르소나`, `7_AOS-DOS`, `8·9_JTBD`)

- **한 문장 정리**  
  > **“시니어·보호자 이중 UX와 의료급 신뢰 포맷을 갖춘 상호운용 허브로,  
  >  3분 온보딩에서 시작해 의사용 1장 요약과 ‘오늘 3가지 행동’을 통해  
  >  진료 수용성과 일상 순응을 동시에 높인다.”**

- **핵심 플로우 (Job → Flow 매핑)**  
  - **F1. 3분 온보딩**  
    - 가입·본인인증·동의·기기/병원 연동을 **3분 내 일괄 처리**  
    - 단계 수·ETA(예상 시간) 표시, 중간 저장, 실패 복구 경로 제공 (N1 대응)  
  - **F2. 1장 의사용 요약 리포트**  
    - 최근 **3/6개월 추세**, 측정 맥락(시간대·기기), 표준 단위 정규화  
    - PDF/인쇄·EMR 첨부를 전제로 한 **의료급 포맷** (C1, “이 수치로 병원에서 인정해 주는가?” 필터 대응)  
  - **F3. 오늘 1~3개 행동 카드**  
    - 행동 중심 홈 화면: “오늘 해야 할 1~3개 행동”만 노출(E2 대응)  
    - 목표 자동 설정, 주간 리듬, 야간 자동 침묵, 오경보 최소화  
  - **F4. 가족 공유 보드**  
    - 시니어/보호자 **각자 계정** + 하나의 공유 보드 (A1, A2)  
    - 초대→동의→역할부여 3단계, 이벤트·알림 뷰 동기화  
  - **F5. 상호운용·의료급 신뢰 레이어**  
    - 멀티벤더 데이터 정합성(워치·혈압계 등), PHR/FHIR 게이트웨이  
    - 동의·위임·감사로그 포함 **의료급 데이터 거버넌스** (신뢰 필터 대응)

### 5) 기존 대안 (Competitor / Substitute)

(출처: `1_포터`, `2_가치사슬 분석`, `5_TAM-SAM-SOM`)

- **전통적 대체재**  
  - 정기 병원 방문, 종합검진, 약국·헬스장, 종이 결과지·수첩·엑셀·카톡·전화  
  - 액티브 시니어는 여전히 **병원 기반 전통 헬스케어에 높은 신뢰**(대체재 위협 강함).

- **글로벌 빅테크/디바이스 허브**  
  - 삼성헬스, Apple Health/HealthKit/Health Records, Google Fitbit + Health Connect  
  - **측정 신뢰/규제 자산(ECG, AFib, 수면무호흡 등)**은 강하지만,  
    시니어 UX·가족 공유·대리접근·행동 코칭은 **부분적/불균형**.

- **국내 B2B2C 패키지형**  
  - SKT-인바이츠, 네이버헬스케어, 카카오헬스케어(병원·지자체·보험 연계)  
  - 문진·동의·예약·결제 위젯, B2B/B2G 패키지 강점  
  - 하지만 **시니어 퍼널·가족 협업·의료급 신뢰 UX**는 들쭉날쭉, MAU 전환 병목 큼.

### 6) 우리가 제공하는 차별적 가치

(출처: `3_KSFs Top5`, `2_가치사슬`, `5_TAM-SAM-SOM`, `7_AOS-DOS`)

- **High-Touch 시니어·보호자 이중 UX**  
  - 처음부터 **시니어 + 보호자를 동시 사용자**로 설계 (Segment 1·2, KSF 1순위)  
  - 가족·케어팀 공유 패널, 역할 분담, 대리 접근 표준화.

- **접근성 퍼스트(A11y) UX**  
  - Extreme 사용자(E1)를 기준으로 **대글자·고대비·음성 안내**를 헌법처럼 설계.  
  - 시력 저하·저디지털 시니어도 “쓸 수 있다”고 느끼는 코어 경험.

- **행동변화 엔진 중심 UX**  
  - 숫자 나열형 앱이 아니라, **“오늘 1~3개 행동”**에 집중하는 행동 코칭 엔진(E2).  
  - AOS·DOS 상 High AOS/High DOS Q1에 위치.

- **3분 온보딩 인프라**  
  - 개별 앱이 아니라, 디지털 헬스 **온보딩 인프라 레이어**를 표준화(N1 전략).  
  - 병원·검진·보험 프로그램의 온보딩을 대신 처리하는 B2B2C 포지션.

- **멀티벤더 상호운용성 & 의료급 신뢰**  
  - 이미 가지고 있는 워치·앱 데이터를 **“전부 가져올 수 있다”**는 Interoperability(신규 진입자 KSF 3순위).  
  - FHIR 게이트웨이·동의·위임·감사로그로 “이 수치로 병원에서 인정해 주는가?” 문제 직격.

### 7) Proof (근거 / 검증 데이터 – 계획)

(출처: `7_AOS-DOS`, `9_JTBD 가상 결과`, `5_TAM-SAM-SOM`)

- **수요·기회 신호(AOS/DOS)**  
  - AOS·DOS Q1 영역: **3분 온보딩, 행동 코칭, 1장 요약 리포트**가  
    High AOS × High DOS 핵심 기회로 확인.  
  - Extreme·Core·보호자 Interview 기반 JTBD 카드에서 동일 Job 반복적으로 출현.

- **파일럿/실험 설계 (JTBD 결과 기반)**  
  - 실험 1: 3분 온보딩 vs 기존 온보딩 → 전환율·1일차 핵심행동 도달률 비교.  
  - 실험 2: “오늘 3가지 행동” 카드 → 주당 목표 달성일, 복약 순응도 효과.  
  - 실험 3: 1장 요약 리포트 → 의사·환자 평점(이해도·결정 속도), 외래 현장 채택률.

- **정량 KPI(초안 – 가상 기준)**  
  - 온보딩 완료율, 첫날 핵심행동 도달률, 월간 1장 리포트 생성률, 가족 대리접근 설정률,  
    복약 순응도 개선, 90일 유지율 등은 `AOS-DOS`·`JTBD`에서 제안한 지표 구조를 그대로 사용.
